Overview

Before you get started

Thank you for purchasing scratch.js on CodeCanyon. If you have any questions that are not answered in this help file, if you've found a bug or think of a feature, please feel free to send me a message with the contact form on my Envato profile page.

Before you get started make sure that you have the 5 following folders:

  • script: this folder contains all the files required to use scratch.js
  • swf: this folder contains the flash file 'scratchcard.swf' required if you want to have a fallback to the html5 canvas tag. You will also find a subfolder called 'source' that contains the source files of the swf.
  • documentation: this folder contains the present documentation
  • graphics: this folder contains all the graphics files used in the examples, feel free to use them in your project if you want to
  • examples: this folder contains three different examples of scratch.js in use, you can check them if you want to see the script in action
  • source: this folder contains the an annoted version of scratch.js in case you need to modify it

Supported browser

All major browsers are supported, including:

  • Internet Explorer 11 and newer (Internet Explorer 9 and newer with Flash fallback)
  • Firefox 3.6 and newer
  • Chrome 2.0 and newer
  • Safari 2.0 and newer
  • Opera 9.0 and newer

scratch.js run on most smartphones:

  • Safari 6.1 and newer for IOS (Iphone and Ipad)
  • Internet Explorer Mobile 10 and newer for Windows Phone
  • Android 4.1 and newer

What's new in this version?

  • some bugs have been fixed, check the changelog for a full list
  • a Flash fallback is available for the browsers not supporting the html5 canvas tag. Check the Installation and Configuration steps for more information.

Getting Started

Step 1 - Installation

The only file you need to upload to your website is in the folder script and is named scratch.min.js. If you want to use the Flash fallback option, you will have to upload to your website the file scratchcard.swf which is in flash folder.

Step 2 - Integration

Once you've uploaded the required file to your server you will need to link it in the head tag of the pages where you want to use sratch.js:

<head>
	<script src="scratch.min.js"></script>
</head>

Step 3 - Configuration

For each scratch card you want to display with scratch.js you will have to set some specific parameters to personnalize their appearance and reactivity. Each instance should have the following properties set:

(all parameters underlined in red are required, the other ones are optional)
  • container: you need to specify a container for the scratch card. For example you can use a div block with an id such as scratchcard that you will identify with document.getElementById('scratchcard').
    Please note that the container you pass as a parameter must have its css display property set to static, otherwise the script will change it to that value.
    Also note that if the container has no width or height specified, the script will use the size of the background image parameter.
  • background: it must indicate a relative or absolute path to the image that you want to be discovered once the foreground image is scratched off. Please note that this image DOES NOT need to follow the same origin policy rule.
  • foreground: it must indicate a relative or absolute path to the image that you want to use as a cover and that will be scrathed off. Please note that due to the same origin policy rule, you can only use an image hosted in the same website than the one where you run the script or on a server using the Cross-Origin Resource Sharing (CORS) (imgur.com, Amazon S3, Google Cloud).
    Alternatively, you can specify an hexadecimal color (for example #7E0 or #9A17C8) instead of an image.
  • percent: this property sets the limit (between 0 and 100) above which the end event will be triggered and the whole scratch card will be automatically scratched off. Please note that a good practice would be to never set this property over 95 as it could be frustrating for your website visitors to have some small and almost hidden pixels left. If nothing is specified the default value is 95.
  • coin: relative or absolute path to an image that will serve as a mouse cursor when the the scratch card will be hovered. If nothing is specified, the default mouse cursor will be set as pointer. If an image is used, it DOES NOT need to follow the same origin policy rule.
  • thickness: set the radius in pixel of the brush that will be used to scratch off the foreground. The higher the value, the higher the size of the brush. Default value is 20
  • counter: specify a function that will be called each time a part of the scratch card is scratched off. The percent of the total discovered area will be used as the function's parameter as well as the current scratch card properties.
  • callback: specify a function that will be called once the end event is triggered. All properties of the current scratch card will be used as the function's parameter.
  • flashPath: it must indicate a relative or absolute path to the swf used as a fallback. Please note that some browsers may not allow that file to be loaded if it is hosted on a https server while your site is http only (and vice versa)
  • flash: this property is a boolean (it must be set to either true or false) that will force the use of Flash instead of using the html5 tag canvas. Please note that this property is for testing purpose only, as Flash is not supported natively on all IOS devices.

Once all options have been set, you have to pass them as an object-parameter to the function createScratchCard. Here is an example:

<script>
	window.onload = function() {
		createScratchCard({
			'container':document.getElementById('scratch2'), 
			'background':'images/demo1-background.png', 
			'foreground':'images/demo1-foreground.png', 
			'percent':90, 
			'coin':'images/coin2.png', 
			'thickness':18,
			'flashPath':'flash/scratchcard.swf'
		});
	}
</script>

Step 4 - Verification

If you followed all the previous steps you page should now display a card ready to be scratched off. If the script does not seem to work, please check the examples in the next step and read the frequently asked question to try to resolve your problem. You can also try to locate and correct the error with web debugging tools such as Firebug (for Firefox) or Chrome Dev Tools (for Chrome).
Ultimately and only if you think that the error is not due to your implementation of scratch.js but is due to a bug in my script, you can send me a message with the contact form on my Envato profile page.

Step 5 - API

Once they have been set, the scratch cards behaviour can be controlled through a basic api. 3 functions are available:

  • lock(): the lock function disables the scratch card (the scratch card cannot be scratched off anymore). The lock function accepts a boolean parameter (true or false) to lock or unlock the card. If no parameter is given, the scratch card will be toggled on or off on each call.
  • restart(): the restart function resets the scratch card to its initial state, even if the scratch card has already been completely scratched off
  • clean(): the clean function scratches off the scratch card without triggering the end event

Those functions are associated with the html container and can be called as follow:

<script>
	document.getElementById('myScratchcard').restart(true);

	function progression(percent, element) {
		if(percent > 50) { element.container.lock(true); }
	}

	function callback(element) {
		element.container.clean();
	}
</script>
  • on line 2 the scratch card's container is selected by using 'getElementById()' and the restart function is called
  • the progression function from line 4 to 6 will lock the container if more than 50% of the card have been scratched off
  • the callback function from line 8 to 10 will clean the container

Examples

Example 1: Basic scratch card

This first example is pretty straight forward and is a good way to understand all basic possibilities of scratch.js

<!DOCTYPE html>
<html>
	<head>
		<meta charset="utf-8" />
		<title>scratch.js - Example 1</title>
		<script type="text/javascript" src="assets/js/scratch.min.js"></script>
		<script>
			function callback(d) { 
				d.container.style.backgroundImage = 'url(assets/images/demo1-end.gif)';
			}
			function percent(p) { document.getElementById('counter').innerHTML = p; }
			window.onload = function() {
				createScratchCard({
					'container':document.getElementById('scratchcard'),
					'background':'assets/images/demo1-background.png',
					'foreground':'assets/images/demo1-foreground.png',
					'percent':95,
					'coin':'assets/images/coin2.png',
					'thickness':18,
					'counter':'percent',
					'callback':'callback'
				});
			}
		</script>
		<style>
			body { text-align: center; }
			#scratchcard { display: block; width: 180px; height: 180px; margin: 40px auto; }
		</style>
	</head>
	<body>
		<div id="scratchcard"></div>
		<span id="counter">0</span><span>%</span>
	</body>
</html>
  • the callback function from line 8 to 10 is called once the whole card has been scratched off. Here we only do one thing : we change the background image to a gif animation in order to give a visual feedback to the visitor that the game has ended
  • the percent function on line 11 update the value of the span tag on line 28, displaying in real time how many percents of the foreground have been scratched off.

Example 2: Complex shape

In some cases you may want to use a shape more complicated than a rectangle or a square to make a scratch card: there is nothing easier, the only thing you have to do is to draw you shape in an image with a transparent background. This example shows some basic ideas that can help you to use the full potential of scratch.js.

<!DOCTYPE html>
<html>
	<head>
		<meta charset="utf-8" />
		<title>scratch.js - Example 2</title>
		<script type="text/javascript" src="assets/js/scratch.min.js"></script>
		<script>
			window.onload = function() {
				createScratchCard({
					'container':document.getElementById('circle'),
					'background':'assets/images/demo2-circle-background.png',
					'foreground':'assets/images/demo2-circle-foreground.png',
					'percent':100,
					'coin':'assets/images/coin2.png',
					'thickness':18,
				});
				createScratchCard({
					'container':document.getElementById('star'),
					'background':'assets/images/demo2-star-background.png',
					'foreground':'assets/images/demo2-star-foreground.png',
				});
				createScratchCard({
					'container':document.getElementById('multiple'),
					'background':'assets/images/demo2-multiple-background.png',
					'foreground':'assets/images/demo2-multiple-foreground.png',
				});
			}
		</script>
		<style>
			body div { display: inline-block; vertical-align: top; }
		</style>
	</head>
	<body>
		<div id="circle"></div>
		<div id="star"></div>
		<div id="multiple"></div>
	</body>
</html>

And that's all you have to do to implement complex shapes! No extra code, scratch.js will automatically differentiate transparent pixels from non-transparent.

Example 3: Choose a card and play

This example demonstrate the full potential of scratch.js. The game works as follow:

  • The player must choose one of the scratch cards
  • When one card has been chosen it can be scratched off, the others are disabled
  • Once the card has been scratched off, the game is over and a message is displayed
  • If the message is clicked, the cards are reseted and shuffled

See a live demo of Example 3

<!DOCTYPE html>
<html>
	<head>
		<meta charset="utf-8" />

		<title>scratch.js - Example 3</title>
		<script type="text/javascript" src="assets/js/scratch.min.js"></script>
		<script>
			var cardSelected = false;
			var cardsArray;
			var instructions;
			var scratchcards;

			var win = 'assets/images/win.png';
			var lose = 'assets/images/lose.png';
			var foreground = 'assets/images/foreground.png';

			function counting(covered, element) {
				if(!cardSelected && covered > 0) {
					for(var i = 0, l = cardsArray.length; i < l; i++) {
						if(!(cardsArray[i].id == element.container.id)) {
							cardsArray[i].lock();
							cardsArray[i].className = 'disabled';
						}
					}
					cardSelected = true;
					instructions.innerHTML = 'Scratch the card off';
				}
			}

			function callback(element) {
				for(var i = 0, l = cardsArray.length; i < l; i++) {
					if(!(cardsArray[i].id == element.container.id)) {
						cardsArray[i].clean();
					}
				}
				var b = element.container.style.backgroundImage;
				if(b.indexOf(win) > -1) {
					instructions.innerHTML = 'You win! Click here to try again!';	
				} else {
					instructions.innerHTML = 'You lose but you can click here to try again.';	
				}
				instructions.addEventListener('click', restartAll, false);
			}
			function restartAll() {
				instructions.removeEventListener('click', restartAll, false);
				for(var i=0, l=cardsArray.length; i<l; i++) {
					cardsArray[i].restart();
					cardsArray[i].className = '';
					isMouseDown = false;
				}
				cardSelected = false;
				shuffle();
				instructions.innerHTML = 'Choose one card';
			}
			function shuffle() {
				for (var i = scratchcards.children.length; i >= 0; i--) {
					scratchcards.appendChild(scratchcards.children[Math.random() * i | 0]);
				}
			}

			window.onload = function() {

				instructions = document.getElementById('instructions');
				instructions.innerHTML = 'Choose one card';

				scratchcards = document.getElementById('scratchcards');

				cardsArray = [ document.getElementById('firstCard'), document.getElementById('secondCard'), document.getElementById('thirdCard') ];
				shuffle();

				createScratchCard({
					'container': cardsArray[0],
					'background': lose,
					'foreground': foreground,
					'coin':'assets/images/coin2.png',
					'percent':95,
					'thickness':18,
					'counter':'counting',
					'callback': 'callback'
				});

				createScratchCard({
					'container': cardsArray[1],
					'background': lose,
					'foreground': foreground,
					'coin':'assets/images/coin2.png',
					'percent':95,
					'thickness':18,
					'counter':'counting',
					'callback': 'callback'
				});

				createScratchCard({
					'container': cardsArray[2],
					'background': win,
					'foreground': foreground,
					'coin':'assets/images/coin2.png',
					'percent':95,
					'thickness':18,
					'counter':'counting',
					'callback': 'callback'
				});

			}
		</script>
		<style>
			body { text-align: center; height: 100%; }
			#instructions { text-align: center; color: #444; font-size: 1.2em; text-transform: uppercase; padding-top: 40px; cursor: pointer; }
			#firstCard, #secondCard, #thirdCard { display: inline-block; vertical-align: middle; margin: 40px;  }
			.disabled { opacity: 0.5; }
		</style>
	</head>
	<body>
		<div id="instructions"></div>
		<div id="scratchcards">
			<div id="firstCard"></div>
			<div id="secondCard"></div>
			<div id="thirdCard"></div>
		</div>
	</body>
</html>
  • from line 9 to 16, all the required variables are set, making them accessible in all functions
  • the counting function from line 18 to 30 is called when one of the cards is scratched off. The condition on line 19 makes sure that the function is only triggered once and the loop on line 20 locks all the cards except the one selected
  • the callback function from line 31 to 44, is called once the chosen card has been scratched off. The loop on line 32 cleans all the other cards and the condition on line 38 check if the player won or lost. On line 43 an event listener is added to allow the player to click on the title to restart the game.
  • the restartAll function from line 45 to 55 resets all the cards and calls the shuffle function
  • the shuffle function from line 56 to 60 shuffles the order of the scratch cards
  • Once the page has been fully loaded, the function from line 62 to 105 will set the scratch cards on the page. There are 2 important lines here: line 69 where we put all the scratch cards' containers in an array, allowing the interactivity between cards (such as clicking one card disables the others) and line 70 where we shuffle the cards before the game starts (so each time the page is reloaded the game is different).

FAQ

  • scratch.js does not seem to work, nothing happens
    • Make sure you have correctly included scratch.js in your page, or try to link the js file with an absolute path. Also make sure that you wait for the page to finish loading before trying to use scratch.js. Your javascript instructions must be enclosed by window.onload: 
      window.onload = function() {
	// your code goes here...
}
  • I want to use an image from another site than mine as a foreground image for a scratch card. Can I bypass the same origin policy rule?
    • It depends, some web servers use the Cross-Origin Resource Sharing (CORS) which in our case allows the use of distant ressources. Servers using CORS include but not limited to: imgur.com, Amazon S3, Google Cloud. If the server does not use CORS, then it is not possible to use a distant image as the same origin policy rule is applied by the browser of your website visitors and thus, cannot be changed.
  • I need to adapt scrath.js to my needs, could you help me?
    • No, unless I think that the feature you want to add could benefit to all user of scratch.js, I will not do free modifications. If you need to edit the script though, check in the source folder as there is an annoted version of it.
  • I need to randomly generate scratch cards, how can I do that?
    • You can find a complementary tutorial here, with a full javascript example and a PHP + javascript example. Please note that the demo files of the second example needs to be uploaded on a php server to work.
